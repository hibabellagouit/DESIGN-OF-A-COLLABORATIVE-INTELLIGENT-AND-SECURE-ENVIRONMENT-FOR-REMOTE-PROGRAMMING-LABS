# Guide étudiant

## Compte GitHub

- À la première connexion, renseignez votre **identifiant GitHub** (sans @).
- Obligatoire pour soumettre et pour le calcul des commits.

## Progression N1–N5

- Chaque niveau : choisir un projet en équipe, soumettre, validation par l’enseignant.
- Le niveau suivant se débloque après validation.

## Soumission

- **Soumission** : fichier, archive ou lien GitHub (un seul mode à la fois).
- Respectez la **date limite** affichée sur le projet si définie.
- **Docker** : `docker-compose.yml` (ou `.yaml`) **à la racine** du projet, du ZIP ou du dépôt GitHub.
- Sur la page Soumission : **checklist** + guide *docker-compose à la racine* (voir aussi [guide-docker-compose-racine.md](./guide-docker-compose-racine.md)).

## Ma note

- Page **Ma note** : détail **50 % note d’équipe** + **50 % participation GitHub (commits)** = note finale /20.

## Classement

- Score global basé sur la progression (niveaux validés) et les notes, selon le barème affiché.
